The YATA bot(s)

The source code of the YATA bot is currently developed and maintained by {{"Kivou"|playerURL:"2000607"|safe}}. Because it is open source, you can encounter different versions controled by different users.{% include "svg/yata.icon.bot.svg" %}

  Official YATA bot

The official YATA bot (later on simply called YATA bot) runs on a single server controled by {{"Kivou"|playerURL:"2000607"|safe}} and is hosted on multiple discord servers owned by different users. It uses both Torn API and YATA database.

  • It is the only one that have access to the YATA database
  • It is the only one vouched for by {{"Kivou"|playerURL:"2000607"|safe}}

You can check if you're dealing with the official bot by checking the list of servers at the bottom of this page and by checking that the bot ID is 623862007434706986 (how to look at discord ID).

  Unofficial YATA bots

There is no control whatsoever on the different versions of the bot. Some might be better, some might be worse, some might be malicious... The only think known is that they have not access to the YATA database and will never have.

Official Torn discord verification

Most of the features will require that your account is official verified on the official Torn discord server. This is the only safe way to link your Torn and Discord account.

API keys management

For some features, the YATA bot needs your API key. And some of these features can be triggered by a tier and display private data on a potentially public channel (see below for details). It will do so only if you give it permission to access it. Even though discord admins don't have access to your key and are only able to use the bot functions, they can play with the roles and the permissions as they wish. If they have malicious intents they can use the bot functions without your knowledge. So, before giving permission to the bot, make sure that you trust the owners of every servers you're in and that are running the YATA bot. Check below the list of features requiring your permission.

Master key for server admins  

The bot requires at least one master key to be able to check discord users identity. Usually the person asking for the bot is putting their but adding others lower the load. No private data is ever pulled from master keys but still the bot requires your permission.

Stock sharing  

If you happen to have the @wssb or @tcb role you're concerned. With your permission, the bot will be able to pull education time left and bank investement time left, respectively, and will make it visible to members of the server (supposed to be shared with @wssb and @tcb members).
