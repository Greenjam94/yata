  API keys management

The bot takes the keys it needs from the YATA database. Before pulling a key it systematically check Torn idendity of the concerned discord member. This verification is made by an API call /user/discord/ checking that Torn and discord identities match. To solve the bootstrap problem, a set of master keys are given to the bot on setup. These keys are only used for this verification procedure. Thus, with this system, you have:

  • to be verified on the official discord server
  • to be registered on the YATA website

to use all the commands of the bot. If you don't, when possible master keys are used as a fallback method. Check the documentation for more details.

For some commands, the YATA bot pulls private informations. And some of these commands can be triggered by a tier, displaying private data on potentially public channels (see below for details). To avoid malicious use of these command a direct message will be sent to you when the data is pulled and displayed.

If your key is set as a master key and you reset it, you just have to logout and log back into YATA with your new key.
